

## Review guidelines

- すべてのレビューコメントは日本語で記述してください。
- コードの問題点、改善提案、賞賛はすべて日本語で行ってください。

## コードコメント規約

- 非自明な判断（性能・設計・互換性・例外対応）はコードコメントに「理由そのもの」を1〜2行で書く。
- **PR ID（`参照: PR #N` 等）はコードコメントに書かない**。`git blame` で辿れる。
- 背景・代替案・却下した選択肢は PR 本文とコミットメッセージに集約する。
- 既存コードに `参照: PR #N` を見つけたら、その行を変更する際に PR 内容を確認して理由を 1〜2 行のコメントに昇格させ、PR 参照は削除する。
- 例外: 外部 issue / ベンダーバグなど永続 URL の参照は可（例: `# workaround for upstream bug: https://...`）。

### fix-flow（Codex 経由の自動修正フロー）への適用

Codex CLI の fix-flow など、自動で issue を修正する運用フローでも上記規約を厳守すること。
過去に「`# ...。参照: PR #N（理由・背景の追跡）`」を機械的に既存コードへ付与する事例があったが、これは禁止する。
