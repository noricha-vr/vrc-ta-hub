# CI テスト安定化パターン

## 外部API依存テストは opt-in にする
- 問題: CI に主要アプリのテストを追加すると、LLM や外部APIを叩くテストまで一緒に走って不安定になる
- 解決: `RUN_EXTERNAL_API_TESTS=1` かつ非ダミーの API キーがある場合だけ実 API テストを実行し、それ以外は `skipTest` または API 層だけモックする
- 教訓: 「鍵が入っているか」だけでなく「ダミー値ではないか」「CIで明示的に opt-in されたか」まで条件に含める

## 初期化を遅延させてルールベース処理を守る
- 問題: 外部APIクライアントを `__init__` で必須初期化すると、ルールベース処理しか使わないテストまで API キー未設定で落ちる
- 解決: クライアント生成を `_get_client()` に分離し、実際に外部APIを呼ぶ経路だけで初期化する
- 教訓: 依存サービスが optional な機能では、初期化タイミングを後ろへ寄せると CI とローカル検証の再現性が上がる
