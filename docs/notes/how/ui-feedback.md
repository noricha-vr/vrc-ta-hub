# UI Feedback パターン

## 長い保存処理はボタン文言だけで終わらせない
- 問題: 記事生成のように保存後もしばらく画面遷移しない処理では、ボタンが沈むだけだとユーザーがフリーズと誤認しやすい。
- 解決: 保存ボタンを `生成しながら保存中…` に切り替えつつ、直下に待機カードを表示して「何をしているか」「このまま待てばいいか」を明示する。
- 教訓: 待機中UIは「状態変化」と「安心材料」を両方出す。ボタンのアニメーションだけより、補助メッセージ付きのカードのほうが離脱防止に効く。

## フロント演出は送信条件と同じ判定で出す
- 問題: 常にローディング演出を出すと、通常保存まで重たく感じる。
- 解決: `generate_blog_article` がONで、かつスライド / PDF / YouTube の入力があるときだけ待機演出を出す。既存データがある編集画面では `data-*` 属性で初期値も渡す。
- 教訓: UX改善でも、サーバー側の実処理条件とズレるとすぐ違和感になる。表示条件はテンプレートに明示的に持たせる。
