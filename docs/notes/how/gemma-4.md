# Gemma 4 運用パターン

## Apple Silicon で常駐させるモデル選定
- 問題: `Gemma 4` は `E2B` / `E4B` / `26B A4B` / `31B` と複数あり、Apple Silicon の大容量RAM環境でどれを常駐対象にすべきか迷いやすい。
- 解決:
  1. 品質最優先なら `google/gemma-4-31B-it` を第一候補にする。Google は `31B Dense` を品質寄りの上位モデルとして位置づけ、`31B` と `26B A4B` に `256K` context を与えている。
  2. 速度と品質のバランスを重視するなら `google/gemma-4-26B-A4B-it` を選ぶ。Google はこのモデルを低レイテンシ寄りとして説明している。
  3. 省メモリやオンデバイス性を優先する場合だけ `google/gemma-4-E4B-it` / `google/gemma-4-E2B-it` を候補に下げる。音声入力は `E2B` / `E4B` のみ対応。
  4. Apple Silicon の大容量統合メモリ環境では、`31B-it` を 4bit から 8bit の量子化で常駐する構成が本命になりやすい。
- 教訓: 大容量RAMのMacでは「入るかどうか」より「品質を取りに行くか、待ち時間を減らすか」で決める方が実用的。1本だけ常駐させるなら `31B-it`、体感速度重視なら `26B A4B-it`。

## Gemma 4 の正式名称と読み方
- 問題: 会話では `Gemma4` と詰めて書かれがちで、正式名称や単一モデルかファミリーかが曖昧になりやすい。
- 解決: 正式名称は `Gemma 4`。単一モデル名ではなく、`E2B` / `E4B` / `26B A4B` / `31B` を含むファミリーとして扱う。
- 教訓: 調査メモや会話ログには `Gemma 4` とスペース入りで書き、必要に応じて `31B-it` などの具体モデルIDまで落とすとズレにくい。
