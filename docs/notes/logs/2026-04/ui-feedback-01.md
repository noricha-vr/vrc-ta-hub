## 記事生成つき保存に待機アニメーションを追加した
- 日付: 2026-04-08
- 関連: なし
- 状況: イベント詳細作成画面とLT申請編集画面で、記事生成を伴う保存中に画面変化が乏しく、ユーザーが不安になりやすかった。
- 問題: 保存処理が長いのに、押下後のフィードバックが弱く、ウィンドウを閉じたくなる体験だった。
- 対応: 共通includeで保存ボタンのローディング状態と待機カードを追加し、記事生成条件に合うときだけ演出を表示するようにした。QAでは submit を一時的に止めて、待機UIの表示と文言ローテーションを確認した。
- → how/ui-feedback.md に知識として追記済み
